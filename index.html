O que é Indexação?

A indexação em banco de dados é uma técnica fundamental para otimizar o tempo de resposta de consultas, permitindo acessar dados de forma mais rápida. Ela cria estruturas auxiliares que facilitam a localização dos registros, sem precisar percorrer toda a tabela.

Sem indexação, uma consulta que envolve milhões de registros pode ser muito demorada, já que o sistema teria que verificar cada linha da tabela uma a uma.

Índice B-Tree

O índice B-Tree (Árvore-B) é uma das estruturas mais usadas para organizar dados de forma hierárquica. Ele é eficiente tanto para buscas por valores exatos quanto para consultas por intervalos. As principais características do B-Tree são:

  • É uma árvore balanceada, garantindo tempo de busca uniforme.
  • Facilita buscas por intervalos de valores, como "maior que" ou "menor que".
  • Permite inserção e exclusão de dados de forma eficiente.

Essa estrutura é amplamente usada em sistemas de banco de dados devido à sua flexibilidade e eficiência para lidar com grandes volumes de dados.

Hash Index

O Hash Index é baseado em uma função de hash que mapeia chaves diretamente para suas localizações no armazenamento. Isso torna as consultas por valores exatos extremamente rápidas. No entanto, ele não é ideal para buscas por intervalos, já que os dados não são organizados de forma ordenada.

  • Alta eficiência para buscas exatas.
  • Não suporta consultas por intervalos.
  • Ideal para colunas com chaves exclusivas, como IDs de usuários.

Esse tipo de índice é útil para casos onde as consultas são feitas por valores exatos, mas pode ser ineficaz em situações que envolvam consultas por intervalo.

Bitmap

O índice Bitmap é ideal para colunas que possuem poucos valores distintos, como status ou flags. Ele usa uma representação compacta, onde cada bit indica a presença ou ausência de um valor em uma linha. Esse tipo de índice é muito eficiente para consultas analíticas que envolvem múltiplos filtros combinados.

  • Eficiência em consultas que envolvem múltiplos filtros.
  • Utiliza menos espaço em colunas com poucos valores repetidos.
  • Ideal para colunas com baixa cardinalidade, como status ou gênero.

No entanto, o índice Bitmap pode não ser ideal para tabelas que passam por muitas atualizações, pois sua manutenção pode ser custosa.
